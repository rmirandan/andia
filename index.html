Andia will allow non-trusting entities to interact with each other

Data Privacy

The Andia protocol allows us entities to interact with each other only when there is a need for compliance, transactions, or KYC without exposing their personally identifiable information.

KYC

Using biometrics, digital data, or other security keys, entities can interact with each other by using limited and anonymous KYC via homomorphic encryption.

Using homomorphic encryption, once the data is verified, they are wiped off and only the hash representation of the document is stored and distributed on the blockchain.

Share data by contract

Our protocol is designed to enable parties to share information securely by a smart contract, re-enforcing the rules of sharing specific information only by a transaction

Pruvyt (™) our first use case

We are able to track customers at our partner retail locations, employing a mobile app (Pruvyt) that uses facial recognition to track customers by only utilizing their biometric vectors (non-identifying information). Using the Pruvyt App in highly regulated markets allows customers to not only be compliant, but also protect customer’s identity through storing the user's encrypted data through our unique storage-and-distribution model. Thus, the Pruvyt App, the first use case for Andia’s Protocol, enables customers and retail locations to interact with each other without the use of identifying data.

HOW IT WORKS

The POI protocol allows for two different entities to share specific data without exposing their entire database. Through smart contracts and off-chain oracles, such entities set the permissions as well as access to data they’d like to share.
